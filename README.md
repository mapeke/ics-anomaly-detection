# ICS Anomaly Detection for Cyber-Physical Security

Undergraduate diploma thesis comparing four anomaly-detection models across two real-world Industrial Control System datasets, evaluated under three time-series-aware metrics.

## Thesis contributions (in progress)

1. **Cross-dataset generalization**: train on one testbed, test on the other.
2. **Time-aware evaluation**: report every model under point-wise, point-adjust, and eTaPR F1 — rankings change.
3. **Per-sensor attribution**: reconstruction-error and attention-based feature attribution.

See [`CLAUDE.md`](CLAUDE.md) for the detailed plan and phase breakdown.

## Running an experiment

```bash
pip install -r requirements.txt
pre-commit install

# One run, 1–3 seeds, parquet row per metric in results/metrics/summary.parquet
python -m experiments.run experiments/configs/baseline_morris_isolation_forest.yaml

# All Phase-1 baselines
for cfg in experiments/configs/baseline_*.yaml; do
    python -m experiments.run "$cfg"
done
```

Each run writes one row per (seed × metric) to `results/metrics/summary.parquet` with a `config_hash` for traceability.

## Repo layout

```
src/
  models/           # AnomalyDetector implementations
  evaluation/       # pointwise, point_adjust, etapr
  config.py         # RunConfig dataclasses + YAML
  data_loader.py    # DatasetBundle (features, labels, attack_ids, split)
  preprocessing.py  # scale_bundle, make_windows, percentile_threshold
  utils.py          # paths, set_seed, save_figure
experiments/
  configs/          # one YAML per run
  run.py            # CLI entry point
tests/              # pytest suite; `-m slow` hits real data
notebooks/          # thin consumers (generated by tools/make_thin_notebooks.py)
results/
  metrics/summary.parquet   # all experiment rows, keyed by config_hash
  figures/                  # PNGs generated by notebooks
```
